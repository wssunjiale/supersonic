# 新增 Superset 图表生成能力

- 需求编号：`需求-03`
- 首次提出：`2026-01-22`
- 最后有效确认：`2026-01-22`
- 时效状态：`有效`
- 实现状态：`进行中`
- 需求主线：`Superset 图表接入`

## 当前定义

- 在保留现有 ECharts 能力的前提下，把 Superset 作为新增图表生成器接入聊天链路，并尽量使用完整的 Superset 图表生态。

## 用户要求

- Superset 是独立部署的，通过 API 与本项目交互。
- Superset 不可用时要回退到现有 ECharts，而不是整体失效。
- 目标不是只支持少量固定图表，而是尽量覆盖 Superset 的图表集合。
- 生成 Superset 图表时，需要明确图表类型、数据源和 SQL，而不是只做静态嵌入。

## 需求边界

- 本需求解决“把 Superset 引进来做图表生成器”。
- 认证通路问题属于 [建立 Superset 双认证连接策略](建立Superset双认证连接策略.md)。
- 嵌入展示细节属于 [完善嵌入式图表展示与交互](完善嵌入式图表展示与交互.md)。
- “推送到看板”入口与目标过滤属于 [将推送入口改为推送到看板](将推送入口改为推送到看板.md)。

## 验收要点

- Superset 可用时，聊天链路能够优先走 Superset 图表生成。
- Superset 不可用、报错或超时时，能自动回退到 ECharts。
- 接入方式能够继续扩展到更多 Superset 图表类型，而不是被写死在少数图表上。

## 归属的 bug / 修复

### 问题 1：Superset 不可用时需要稳定回退

- 现象：一旦 Superset API 失败或超时，整条绘图链路容易被卡死。
- 根因：新增图表生成器时，如果没有先收紧失败边界，回退逻辑就会变成偶发行为。
- 处理：把“失败或超时后自动回退 ECharts”写成显式能力，而不是临时兜底。
