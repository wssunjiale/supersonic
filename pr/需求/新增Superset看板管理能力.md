# 新增 Superset 看板管理能力

- 需求编号：`需求-10`
- 首次提出：`2026-02-10`
- 最后有效确认：`2026-02-10`
- 时效状态：`有效`
- 实现状态：`进行中`
- 需求主线：`Superset 看板管理界面`

## 当前定义

- 新增一个顶级 Superset 看板管理页面，用于管理手工创建的看板，而不是聊天链路自动生成的单图临时看板。

## 用户要求

- 页面放在顶级目录。
- 管理员可以管理全部看板；普通用户只能管理自己的看板。
- 支持新建空看板，并同步在 Superset 中创建。
- 支持查看看板，以 embed 方式在新标签页打开。
- 支持删除看板，同时删除 Superset 中的对应看板。
- 支持编辑看板，跳到 Superset 的 dashboard edit 页面。
- 列表里必须过滤掉自动生成的单图临时看板。

## 需求边界

- 本需求关注“手工看板的生命周期管理”。
- 聊天消息里的“推送到看板”入口与目标过滤属于 [将推送入口改为推送到看板](将推送入口改为推送到看板.md)。
- 临时单图看板仍然可以存在，但不应被这个管理页纳入正式管理范围。

## 验收要点

- 顶级管理页可以列出手工看板。
- 新建、查看、删除、编辑链路都能打通到 Superset。
- 自动生成的单图临时看板不会出现在该页面的列表中。
- 权限边界清晰：管理员看全量，普通用户看自己的。

## 归属的 bug / 修复

- 暂无需要长期保留的独立问题卡。
