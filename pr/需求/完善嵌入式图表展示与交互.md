# 完善嵌入式图表展示与交互

- 需求编号：`需求-05`
- 首次提出：`2026-02-02`
- 最后有效确认：`2026-02-07`
- 时效状态：`有效`
- 实现状态：`进行中`
- 需求主线：`Superset 嵌入展示`

## 当前定义

- 修复 Superset dashboard / chart 在测试页和聊天消息中的嵌入展示问题，包括自动获取嵌入信息、尺寸、自适应和基础交互体验。

## 用户要求

- `superset-embed-test.html` 不应要求人工输入 token，应自动获取。
- 选择 dashboard 后，嵌入需要的标识信息应尽量自动补齐。
- 聊天消息中的嵌入看板不能只是在测试页可用，真实消息中也要正确显示。
- 看板区域不能过小，尺寸需要更符合消息容器场景。

## 需求边界

- 本需求只处理“已经有可嵌入资源之后，如何稳定展示它”。
- 看板列表获取和目标过滤属于 [将推送入口改为推送到看板](将推送入口改为推送到看板.md)。
- 认证通路属于 [建立 Superset 双认证连接策略](建立Superset双认证连接策略.md)。

## 验收要点

- 测试页可以自动获取 token 和嵌入所需信息。
- 消息体内能稳定显示嵌入内容，而不是只在测试页成功。
- 嵌入区域尺寸、自适应和基础交互体验符合真实聊天场景。

## 归属的 bug / 修复

### 问题 1：测试页嵌入信息依赖手填

- 现象：测试页需要人工填 token 或缺少可直接嵌入的必要字段。
- 根因：测试页和后端接口之间缺少统一的嵌入元信息封装。
- 处理：由 Supersonic 后端统一提供 token 与 dashboard 关键元信息。

### 问题 2：嵌入区域过小，自适应体验差

- 现象：消息内或测试页里的嵌入区域过小，无法根据容器稳定调整高度。
- 根因：原实现过度依赖固定高度，没有同时考虑消息容器和 iframe 内容高度。
- 处理：把消息容器可用区域作为主依据，再结合 iframe 内容高度做兜底。
