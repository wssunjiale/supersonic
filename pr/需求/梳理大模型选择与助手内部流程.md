# 梳理大模型选择与助手内部流程

- 需求编号：`需求-14`
- 首次提出：`2026-01-27`
- 最后有效确认：`2026-01-27`

- 实现状态：`已落地`
- 需求主线：`助手与模型认知梳理`

## 当前定义

- 梳理当前系统在“配置了多个大模型时怎么选模型”，以及各种智能助手内部流程中哪些步骤真正调用了大模型。

## 用户要求

- 如果配置了两个大模型，要明确程序最终会选哪个。
- 要指出哪些步骤实际用了大模型，哪些没有。
- 要梳理不同“助手”的内部工作流程，避免后续改造时误判调用边界。

## 需求边界

- 这条需求主要是认知与流程澄清。
- 不直接要求改造模型路由策略本身，也不自动延伸到提示词或 Schema Mapping 调整。

## 验收要点

- 能明确多模型配置下的选择逻辑。
- 能列出各助手链路中真正使用 LLM 的环节。
- 形成可复用的内部流程认知，而不是零散回答。

## 归属的 bug / 修复

- 暂无需要长期保留的独立问题卡。
