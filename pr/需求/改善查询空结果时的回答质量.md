# 改善查询空结果时的回答质量

- 需求编号：`需求-18`
- 首次提出：`2026-03-06`
- 最后有效确认：`2026-03-06`
- 时效状态：`有效`
- 实现状态：`未判定`
- 需求主线：`问答结果总结`

## 当前定义

- 当查询结果为空时，不再输出带占位符、伪数据或模板残片的“假总结”，而是输出合理、诚实、可解释的空结果回答。

## 用户要求

- 空结果时的回答不能再像“把模板没填完就发给用户”。
- 回答应体现“没有查到数据”这一事实，而不是假装已经分析出了若干产品和金额。
- 输出要合理、有效，而不是刻板套模板。

## 需求边界

- 本需求解决的是“空结果时怎么说”。
- 为什么会查不到数据、为什么 SQL 错误、为什么字段选错，不归入本需求。

## 验收要点

- 空结果时不再输出占位符和伪造数据。
- 回答能明确说明“当前没有查到满足条件的数据”。
- 在不臆造数据的前提下，给出合理的解释或下一步建议。

## 归属的 bug / 修复

### 问题 1：空结果时仍然套用正常分析模板

- 现象：回答中直接把“产品A”“请在此处插入金额”等占位符原样输出给用户。
- 根因：结果总结模板没有针对空结果做分支处理，导致模板残片直接泄漏。
- 处理：为空结果单独设计总结路径，不再套用普通数据分析模板。
