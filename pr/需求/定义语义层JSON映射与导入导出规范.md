# 定义语义层 JSON 映射与导入导出规范

- 需求编号：`需求-19`
- 首次提出：`2026-02-10`
- 最后有效确认：`2026-02-10`
- 时效状态：`有效`
- 实现状态：`未判定`
- 需求主线：`语义层导入导出`

## 当前定义

- 设计一套“指标体系设计文档 -> JSON -> Supersonic Headless BI API”的语义层交换格式，同时支持从系统现有语义层反向导出同样格式的 JSON。

## 用户要求

- JSON 只保留业务必要字段，不做无意义的字段堆砌。
- JSON 需要覆盖语义层中的数据域、数据集、数据模型、数据库等定义。
- 未来会有程序解析该 JSON，并通过后端 API 导入语义层。
- 反过来也要能把系统内现有语义层导出成同样格式。
- 导入时如果发现系统里已有定义，要按 `name` 匹配并覆盖合并。
- 一旦出现异常或错误，要整体失败退出，并打印完整错误信息。

## 需求边界

- 本需求先解决“交换格式和导入导出语义”。
- 具体导入程序实现可以后续再做，但格式设计和合并规则需要先定下来。

## 验收要点

- JSON 格式覆盖语义层关键实体，且字段粒度控制在“业务必要字段”。
- 导入导出使用同一套格式。
- 合并规则明确为“按 `name` 匹配、导入覆盖”。
- 错误处理策略明确为“整体失败退出，不静默吞错”。

## 归属的 bug / 修复

### 问题 1：导入出错时如何处理没有被提前定义

- 现象：用户明确要求一旦导入过程中出现异常或错误，就直接退出并打印完整错误信息。
- 根因：如果格式设计不先约束失败策略，后续实现很容易变成局部成功、局部失败的不可控状态。
- 处理：把“整体失败退出、完整打印错误”写进规范本身。
