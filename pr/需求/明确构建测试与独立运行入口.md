# 明确构建、测试与独立运行入口

- 需求编号：`需求-02`
- 首次提出：`2026-02-03`

- 需求主线：`基础环境与运行入口`

## 当前定义

- 明确 Supersonic 二开仓库的正确构建命令、测试入口、IDE 运行入口，以及独立运行时的关键配置位置。

## 用户要求

- 不再让后续开发因为“测试跑不起来”“入口找不到”“IDE 不能运行”而反复卡住。
- 测试与构建要给出真实可用的入口，而不是理论上的命令。
- 要说明 `StandaloneLauncher`、配置文件和运行时环境之间的对应关系。

## 需求边界

- 本需求只解决“怎么正确构建、测试、启动”。
- Superset 同步 runner、配置扫描和离线验收，属于 [建立 Superset 语义同步链路与离线验收运行器](建立Superset语义同步链路与离线验收runner.md)。
- 一般业务 bug 不因为“启动时报错”就自动归入本需求。

## 验收要点

- 能给出根目录可执行的构建与测试命令。
- IDE 能定位并运行 `StandaloneLauncher`。
- 能指出独立运行时需要关注的配置文件、环境变量和 JDK 版本。

## 归属的 bug / 修复

### 问题 1：从子模块直接跑测试经常缺依赖

- 现象：在子模块目录直接跑测试时，经常出现依赖不全、模块未先构建的问题。
- 根因：多模块工程没有先在仓库根目录统一构建，局部模块直接执行容易缺前置产物。
- 处理：测试与构建应以仓库根目录为主执行，必要时使用 `-am` 补齐依赖模块。

### 问题 2：`Run Java` 找不到 `StandaloneLauncher`

- 现象：IDE 无法识别或运行 `StandaloneLauncher`。
- 根因：根目录导入方式、JDK 21、Maven 模型识别和子模块 Java 版本配置没有对齐。
- 处理：明确以仓库根目录导入项目，并把 `StandaloneLauncher` 作为主运行入口来校验。
